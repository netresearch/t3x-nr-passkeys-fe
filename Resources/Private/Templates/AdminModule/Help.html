FE Passkey Management Help

This guide explains how to roll out passkeys to your frontend users, handle recovery scenarios, and understand enforcement levels.

Rolling out frontend passkeys

A phased rollout lets you introduce passkeys gradually, giving users time to register their devices before enforcement becomes mandatory.

Step 1 — Install the extension

Install netresearch/nr-passkeys-fe via Composer and activate it in the Extension Manager. Run the database analyser to create the tx_nrpasskeysfe_credential table.

Step 2 — Add the plugins to your site

Add the FE Passkey Login, FE Passkey Management, and (optionally) FE Passkey Enrollment plugins to the appropriate pages of your site. Configure TypoScript to point the plugins at the correct frontend user storage folder.

Step 3 — Set groups to Encourage

On the Dashboard tab, change a frontend user group's enforcement level to Encourage. Users in that group will see a non-blocking banner on the site prompting them to register a passkey.

Start with a pilot group to validate the workflow before widening the rollout to all frontend users.

Step 4 — Set groups to Required

When you are confident the process works, change the group to Required. Users who have not yet registered a passkey will be directed to the enrollment page after password login. A configurable grace period gives them time to register before the requirement becomes strict.

Step 5 — After the grace period expires

Once the grace period ends, users in Required groups must register a passkey before they can access the site. Users in Enforced groups have no grace period at all.

Step 6 — Monitor adoption

Use the Dashboard tab to track overall adoption rates and per-group statistics. Use the Passkey Lookup panel to inspect an individual user's registered credentials.

Recovery procedures

When a frontend user loses access to their authenticator device, follow these steps:

1. User reports lost device

The user contacts you (or a site admin) and reports which device or authenticator is no longer available.

2. Admin revokes affected passkey(s)

In Web > FE Passkeys > Dashboard, enter the user's UID in the Passkey Lookup panel and revoke the affected credential(s). If you're unsure which credential matches the lost device, look at the creation date and label.

Alternatively, open the user's fe_users record in the backend list module. The Passkeys inline element in the record shows all registered credentials and allows deletion.

3. User has zero passkeys and enforcement is active

If the user's group is set to Required or Enforced and the user now has zero registered passkeys, the next password-based login will direct them to the enrollment page. They must register a new passkey on their replacement device before they can continue.

4. User cannot log in at all

If the user has no working authenticator and is rate-limited (too many failed attempts), use the Unlock user button in the Passkey Lookup panel to reset the lockout.

If they also cannot complete password-based login, the administrator can temporarily lower the group enforcement level to Encourage or Off so the user can log in with a password and re-register a passkey.

Remember to restore the group's enforcement level after the user has re-registered a passkey. Leaving enforcement lowered indefinitely weakens your site's security.

Enforcement levels

Level Behaviour
Off Passkeys are available but not promoted. No banner or redirect.
Encourage A dismissible banner prompts users without passkeys to register one. They can continue using the site with a password.
Required Users without a passkey are redirected to the enrollment page after password login. They may skip during the grace period; after it expires they must register.
Enforced Same as Required but without any grace period. Users must register a passkey immediately.
When a user belongs to multiple groups, the strictest enforcement level takes effect. For example, if one group is Off and another is Required, the user sees Required.
